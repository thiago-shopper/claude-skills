# event template

## When to use
One-time gathering: conference, meetup, workshop, hackathon, unconference, retreat, dinner, anything with a fixed date and a guest list.

## Default audience
Invited attendees of a defined size (estimate from the draft if given, else propose 30–80 and label as Assumption). Co-organisers and venue / vendor partners as secondary stakeholders.

## Default timeline + milestones
- Horizon: **12 weeks** lead time, then event day + a debrief week.
- Milestones (E = event day):
  1. E-12 weeks — concept, audience, and approximate size locked. Date held. Budget envelope set.
  2. E-8 weeks — venue confirmed; save-the-date sent to confirmed invitees / opened sign-ups.
  3. E-4 weeks — programme / agenda locked; speakers or facilitators confirmed; logistics (catering, AV, accessibility) booked.
  4. E-1 week — final headcount + dietary / accessibility info collected; day-of runbook circulated to all co-organisers.
  5. E day — the event itself.
  6. E + 1 week — debrief, attendee feedback survey out, follow-up content (if any) shipped.

## Default resources
- One primary organiser (the user) + 1–2 informal helpers.
- A modest budget — assume self-funded or sponsor-funded unless the draft says otherwise.
- Off-the-shelf event tooling (Luma / Eventbrite / Partiful + a shared doc for logistics).
- No paid PR / paid promotion by default.

## Likely risks (template-derived)
- Venue falls through close to the date → mitigation: hold a backup venue at E-8 weeks; confirm at E-4 weeks.
- Sign-up no-show rate higher than expected → mitigation: assume 20–30% no-show; over-invite or charge a small deposit if appropriate.
- Programme is too packed (no breathing room) → mitigation: schedule explicit unstructured time; cap content blocks at 60 minutes.
- Accessibility / dietary / safety issues surface day-of → mitigation: collect requirements at E-1 week; brief co-organisers; have a written escalation contact.
- Post-event drop-off — no follow-through with attendees → mitigation: prewrite the E+1 follow-up email at E-1 week.

## Default success criteria
- Leading metric: confirmed attendees by E-1 week ≥ target headcount (set from draft, else propose target and label as Assumption).
- Lagging metric: post-event survey returns Net Promoter / "would attend again" ≥ a stated threshold (e.g. ≥ 8/10 average, response rate ≥ 40%).

## Preferred output-format default
A week-by-week T-minus calendar (E-12 → E day → E+1 week), a day-of runbook hour-by-hour, a vendor + logistics checklist, a risk register, and a follow-up plan for the week after. Flag any assumption about audience size, budget, or venue type I should confirm before approving.
