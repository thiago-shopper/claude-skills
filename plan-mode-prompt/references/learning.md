# learning template

## When to use
Self-directed study / skill acquisition: learn a language, framework, domain, profession, or specific technical skill. Roadmaps, bootcamps, courses, "I want to master X by Y".

## Default audience
The learner themself. A reviewer or accountability partner if the draft mentions one.

## Default timeline + milestones
- Horizon: **90 days**, three 30-day blocks.
- Milestones:
  1. Day 10 — fundamentals warm-up complete; first end-to-end "hello world" project shipped.
  2. Day 30 — first non-trivial project complete; written reflection on what's still confusing.
  3. Day 60 — second non-trivial project; tackled at least one hard sub-topic the user was avoiding.
  4. Day 90 — capstone project shipped publicly (repo, post, demo) + one-page self-assessment vs. initial goal.

## Default resources
- Self-study, solo.
- ≤ 8 hours / week of study time.
- Free / off-the-shelf learning materials (one canonical book or course + free online docs).
- Existing tooling (no new hardware / paid subscription) unless the draft says so.

## Likely risks (template-derived)
- Tutorial trap — endlessly consuming, never building → mitigation: every Sunday, ship a small artifact (commit, post, exercise) tied to that week's reading; no exceptions.
- Curriculum sprawl — picking up too many resources at once → mitigation: pick ONE canonical resource at Day 0; only after Day 30 may a second be added.
- Avoiding hard sub-topics → mitigation: list the 3 most-feared sub-topics on Day 1; require one of them tackled in Days 30–60.
- Loss of momentum at the Day 30 plateau → mitigation: schedule a public progress post (or a check-in with a peer) for Day 30 so it's harder to ghost.
- Goal drift — narrowing or widening mid-study → mitigation: revisit Day-0 goal text at each 30-day checkpoint and write "still aligned / pivoting because…" explicitly.

## Default success criteria
- Leading metric: at least one shipped artifact per week through week 12 (12 artifacts total, even tiny ones).
- Lagging metric: capstone project complete by Day 90 + written self-assessment that judges fluency on the original goal (e.g. "I can build a small CRUD app in Rust without copying from a tutorial").

## Preferred output-format default
A 90-day study plan rendered week-by-week (12 weeks), a list of "ship something every Sunday" artifacts (week 1 → week 12), a feared-topic roster with which 30-day block each one belongs to, and a capstone-project spec. Flag any assumption about prior background or available time I should confirm.
