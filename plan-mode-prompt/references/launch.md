# launch template

## When to use
Product / service launch, GTM motion, marketing push, public release, or any "we will tell the world about X on date Y" effort.

## Default audience
- **Primary**: target customers / users in the segment the product serves.
- **Secondary**: existing customers / community whose attention you'll borrow on day one.
- **Tertiary**: press / analysts / partners, if relevant.

## Default timeline + milestones
- Horizon: **6 weeks** from kickoff to launch day, then **+2 weeks** of post-launch operations.
- Milestones (T = launch day):
  1. T-6 weeks — positioning, ICP, and one-line message locked.
  2. T-4 weeks — landing page live; waitlist or pre-orders collecting; messaging tested with 5 target users.
  3. T-2 weeks — full asset set ready (page, demo, screenshots, FAQ, support docs); friends-and-family soft-launch behind a flag.
  4. T-0 — public launch on the chosen primary channel.
  5. T+2 weeks — post-launch review: what landed, what flopped, what to keep doing.

## Default resources
- Founder/owner running the launch personally.
- Existing audience (personal network, social, mailing list) as the day-zero distribution.
- Off-the-shelf landing-page + email tooling.
- No paid acquisition budget by default.

## Likely risks (template-derived)
- Message resonates with the team but not the market → mitigation: validate the headline + subhead with 5 target users at T-4 weeks; revise if < 3 of 5 immediately "get it".
- Launch-day infra cannot handle the traffic spike → mitigation: load-test at T-1 week; have a static fallback page ready.
- One channel underperforms (e.g. social) → mitigation: line up at least two independent launch channels (e.g. social + a launch site or partner cross-post).
- Post-launch silence — no follow-up content → mitigation: pre-plan the T+1, T+3, T+7 follow-up posts before launch day.
- Support load on launch day overwhelms solo founder → mitigation: pre-write 10 FAQ answers; set support expectations on the landing page.

## Default success criteria
- Leading metric: ≥ X waitlist signups by T-2 weeks (set X based on audience size in draft, else leave for the user to define).
- Lagging metric: ≥ Y trials / signups / paying conversions in the first 14 days post-launch (set Y based on draft, else leave for the user).

## Preferred output-format default
A T-minus calendar (week-by-week), the launch-day runbook (hour-by-hour for T-0 itself), a channel plan with assets per channel, a risk register with mitigations, and a definition of "successful launch" with two numeric thresholds. Flag any positioning assumptions I should validate before approving.
