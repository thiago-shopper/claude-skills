# generic template

## When to use
Fallback when no other plan-type keyword matched. Use for personal goals, exploratory projects, mixed-domain ideas, anything genuinely uncategorisable.

## Default audience
The user themself, plus 1–2 trusted advisors they will share the plan with. No external audience by default.

## Default timeline + milestones
- Horizon: **90 days**, reviewed monthly.
- Milestones:
  1. Day 7 — scope locked + first concrete output produced.
  2. Day 30 — first review checkpoint; adjust direction.
  3. Day 60 — second review; decide whether to extend, pivot, or stop.
  4. Day 90 — finished or knowingly extended.

## Default resources
- Solo effort.
- Off-the-shelf tooling only.
- No dedicated budget.
- Existing personal network / no new hires.

## Likely risks (template-derived)
- Scope creep — keeps expanding past Day-30 checkpoint → mitigation: lock written scope at Day 7 and require explicit edit to change.
- Loss of motivation around the messy middle (Day 30–60) → mitigation: schedule the Day-30 review with someone else, not solo.
- Hidden dependencies surfacing late → mitigation: write a "what would have to be true" list at Day 7 and revisit Day 30.

## Default success criteria
- Leading metric: at least one concrete artifact produced by Day 7 (write what it is at Day 0).
- Lagging metric: a written self-assessment at Day 90 saying whether the goal was hit, with one objective measure.

## Preferred output-format default
Produce a 90-day plan as a weekly outline (12 rows), plus a written "scope-lock" paragraph for Day 7, plus a one-page risk register. Surface assumptions I have not explicitly confirmed.
